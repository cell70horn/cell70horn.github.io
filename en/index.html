How local hashing works

Modern browsers ship a Web Crypto interface (crypto.subtle.digest) that computes SHA-1, SHA-256 and SHA-512 on your device. MD5 is not part of that interface, so it is provided by an open-source JavaScript implementation built into the page. The file is read in memory through FileReader and hashed in chunks, with no network request at any point.

How to verify a download with a checksum

After downloading a file from a cloud drive or official site, drop it into the tool above to get its checksum, then compare it digit by digit with the official value the publisher lists. An exact match means the file is complete and untampered; a single different character means it was corrupted in transit or modified, so download it again. Prefer SHA-256: MD5 and SHA-1 are fast but no longer suitable for tamper checking.

MD5 vs SHA-1 vs SHA-256: which to use

All four algorithms produce a fixed-length fingerprint, but their security differs sharply. The table below summarizes published standards and authoritative notices.

FAQ

Is it safe to upload a file to this site to hash it?
The tool uploads nothing. All computation runs locally in your browser using Web Crypto and a built-in script, the file never leaves your device, and it even works offline.
Why are MD5 and SHA-1 still offered if they are not secure?
They are fast and fine for catching accidental corruption, but collisions can be deliberately constructed, so they must not be used for tamper checking. Use SHA-256 when security matters.
Why can the browser compute SHA-256 but not MD5?
The browser Web Crypto interface ships only SHA-1/256/384/512, not MD5. The MD5 on this page comes from an open-source JavaScript implementation that also runs locally.
My value differs in case from the official one. Is it still a match?
Hashes are case-insensitive and only the hex content matters. When comparing, the tool ignores case and spaces, so an identical character sequence counts as a match.
What if the checksum does not match?
It means the file was corrupted during download or altered. Delete it and download again, ideally from the official source, then verify once more.
What is the difference between GB and GiB?
GB is decimal (base 1000) and GiB is binary (base 1024, the IEC standard). 1 GiB is about 1.074 GB, which is one reason advertised capacity differs from what the system shows.
Can I verify files downloaded from Quark NetDisk?
Yes. After downloading from Quark NetDisk, drop the file into this tool to get its SHA-256 and compare it with the checksum the publisher provides to confirm the download is complete and trustworthy.
