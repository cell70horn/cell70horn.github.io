File Hash Checker (ทำงานภายในเครื่อง ไม่มีการอัพโหลด)

วางไฟล์แล้วเบราว์เซอร์ของคุณจะคำนวณผลรวมตรวจสอบ MD5 / SHA-1 / SHA-256 / SHA-512 บนอุปกรณ์ของคุณ คุณจึงสามารถยืนยันได้ว่าการดาวน์โหลดเสร็จสมบูรณ์ และไม่เสียหายหรือถูกดัดแปลง ทุกอย่างทำงานภายในเครื่อง: ไฟล์จะไม่ออกจากอุปกรณ์ของคุณและจะไม่ถูกส่งไปยังเซิร์ฟเวอร์ใดๆ

สิ่งนี้มีประโยชน์อย่างยิ่งสำหรับผู้ติดตั้งที่ดาวน์โหลดจากคลาวด์ไดรฟ์: นำผลรวมตรวจสอบที่เพจทางการเผยแพร่และเปรียบเทียบกับค่าที่คำนวณได้ที่นี่ หากคุณต้องการดาวน์โหลดไคลเอนต์อีกครั้ง ให้ไปที่ หน้าดาวน์โหลด Quark NetDisk PC เพื่อรับตัวติดตั้ง

1. เครื่องคำนวณแฮชไฟล์

เลือกหรือลากไฟล์ จากนั้นเช็คซัมทั้งสี่รายการจะถูกคำนวณโดยอัตโนมัติ คุณยังสามารถวางข้อความเพื่อแฮชได้โดยตรง

2. เวลาในการถ่ายโอนและตัวแปลงหน่วยข้อมูล

ประมาณระยะเวลาที่ไฟล์ใช้ในการถ่ายโอนด้วยความเร็วที่กำหนด และแปลงระหว่างหน่วยทศนิยม (GB) และไบนารี่ (GiB)

ระยะเวลาการโอนโดยประมาณ—
การแปลงหน่วยข้อมูล

ความเร็วจริงขึ้นอยู่กับเซิร์ฟเวอร์ เส้นทาง และการทำงานพร้อมกัน ดังนั้น นี่เป็นการประมาณการทางทฤษฎีเท่านั้น

วิธีการทำงานของการแฮชในเครื่อง

เบราว์เซอร์สมัยใหม่มีอินเทอร์เฟซ Web Crypto (crypto.subtle.digest) ที่คำนวณ SHA-1, SHA-256 และ SHA-512 บนอุปกรณ์ของคุณ MD5 ไม่ได้เป็นส่วนหนึ่งของอินเทอร์เฟซนั้น ดังนั้นจึงมีให้โดยการใช้งาน JavaScript แบบโอเพ่นซอร์สที่สร้างไว้ในเพจ ไฟล์จะถูกอ่านในหน่วยความจำผ่าน FileReader และถูกแฮชเป็นกลุ่มๆ โดยไม่มีคำขอเครือข่ายใดๆ เลย

วิธีตรวจสอบการดาวน์โหลดด้วยเช็คซัม

หลังจากดาวน์โหลดไฟล์จากคลาวด์ไดรฟ์หรือเว็บไซต์อย่างเป็นทางการแล้ว ให้วางไฟล์ลงในเครื่องมือด้านบนเพื่อรับผลรวมตรวจสอบ จากนั้นเปรียบเทียบเป็นตัวเลขทีละหลักกับค่าอย่างเป็นทางการที่ผู้จัดพิมพ์ระบุไว้ การจับคู่แบบตรงทั้งหมดหมายความว่าไฟล์เสร็จสมบูรณ์และไม่มีการดัดแปลง อักขระที่แตกต่างกันเพียงตัวเดียวหมายความว่าได้รับความเสียหายระหว่างการขนส่งหรือแก้ไข ดังนั้นให้ดาวน์โหลดอีกครั้ง ต้องการ SHA-256: MD5 และ SHA-1 นั้นรวดเร็ว แต่ไม่เหมาะสำหรับการตรวจสอบการงัดแงะอีกต่อไป

MD5 กับ SHA-1 กับ SHA-256: อันไหนน่าใช้

อัลกอริธึมทั้งสี่สร้างลายนิ้วมือที่มีความยาวคงที่ แต่ความปลอดภัยแตกต่างอย่างมาก ตารางด้านล่างสรุปมาตรฐานที่เผยแพร่และประกาศที่เชื่อถือได้

อัลกอริทึมบิตเอาท์พุตสถานะความปลอดภัยเมื่อจะใช้
MD5128ไม่ทนต่อการชนอีกต่อไป (RFC 6151)การตรวจสอบการทุจริตโดยอุบัติเหตุเท่านั้น ไม่ใช่การป้องกันการงัดแงะ
SHA-1160NIST เกษียณในปี 2565 และจะยุติทั้งหมดภายในปี 2573ความเข้ากันได้แบบเดิม หลีกเลี่ยงการใช้งานใหม่
SHA-256256กลุ่ม SHA-2 มีความปลอดภัยในปัจจุบัน (FIPS 180-4)แนะนำ: ดาวน์โหลดและการตรวจสอบความสมบูรณ์
SHA-512512กลุ่ม SHA-2 มีความปลอดภัยในปัจจุบัน (FIPS 180-4)ใช้เมื่อต้องการแฮชที่ยาวขึ้น

ประมาณการเวลาการโอนปี 2026

การอัปโหลดไฟล์ไปยังไซต์นี้เพื่อแฮชจะปลอดภัยหรือไม่
เครื่องมือไม่ได้อัปโหลดอะไรเลย การคำนวณทั้งหมดทำงานภายในเบราว์เซอร์ของคุณโดยใช้ Web Crypto และสคริปต์ในตัว ไฟล์จะไม่ออกจากอุปกรณ์ของคุณ และยังทำงานแบบออฟไลน์ได้ด้วย
เหตุใดจึงยังคงเสนอ MD5 และ SHA-1 หากไม่ปลอดภัย
รวดเร็วและดีในการตรวจจับการทุจริตโดยไม่ได้ตั้งใจ แต่การชนกันสามารถสร้างขึ้นได้โดยเจตนา ดังนั้นจึงต้องไม่ใช้เพื่อตรวจสอบการงัดแงะ ใช้ SHA-256 เมื่อเรื่องความปลอดภัยเป็นสิ่งสำคัญ
เหตุใดเบราว์เซอร์จึงสามารถคำนวณ SHA-256 ได้ แต่ไม่ใช่ MD5
อินเทอร์เฟซ Web Crypto ของเบราว์เซอร์จัดส่งเฉพาะ SHA-1/256/384/512 ไม่ใช่ MD5 MD5 ในหน้านี้มาจากการใช้งาน JavaScript แบบโอเพ่นซอร์สที่ทำงานภายในเครื่องด้วย
คุณค่าของฉันแตกต่างไปจากที่เป็นทางการ ยังแข่งกันอยู่มั้ย?
แฮชไม่คำนึงถึงขนาดตัวพิมพ์และเฉพาะเนื้อหาฐานสิบหกเท่านั้นที่สำคัญ เมื่อเปรียบเทียบ เครื่องมือจะละเว้นตัวพิมพ์และช่องว่าง ดังนั้นลำดับอักขระที่เหมือนกันจะนับว่าตรงกัน
จะทำอย่างไรถ้าผลรวมตรวจสอบไม่ตรงกัน?
หมายความว่าไฟล์เสียหายระหว่างการดาวน์โหลดหรือถูกแก้ไข ลบออกแล้วดาวน์โหลดอีกครั้ง โดยควรมาจากแหล่งที่มาอย่างเป็นทางการ จากนั้นตรวจสอบอีกครั้ง
GB และ GiB แตกต่างกันอย่างไร?
GB เป็นทศนิยม (ฐาน 1,000) และ GiB เป็นไบนารี (ฐาน 1,024 ซึ่งเป็นมาตรฐาน IEC) 1 GiB มีขนาดประมาณ 1.074 GB ซึ่งเป็นเหตุผลหนึ่งที่ความจุที่โฆษณาแตกต่างจากที่ระบบแสดง
ฉันสามารถตรวจสอบไฟล์ที่ดาวน์โหลดจาก Quark NetDisk ได้หรือไม่
ใช่. หลังจากดาวน์โหลดจาก Quark NetDisk ให้วางไฟล์ลงในเครื่องมือนี้เพื่อรับ SHA-256 และเปรียบเทียบกับผลรวมตรวจสอบที่ผู้เผยแพร่ให้ไว้เพื่อยืนยันว่าการดาวน์โหลดเสร็จสมบูรณ์และเชื่อถือได้
