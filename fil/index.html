File Hash Checker (tumatakbo nang lokal, walang pag-upload)

Maglagay ng file at kino-compute ng iyong browser ang MD5 / SHA-1 / SHA-256 / SHA-512 checksum nito mismo sa iyong device, para makumpirma mong kumpleto na ang pag-download at hindi nasira o na-tamper. Lahat ay tumatakbo nang lokal: ang file ay hindi kailanman umaalis sa iyong device at hindi kailanman ipapadala sa anumang server.

Ito ay lalong madaling gamitin para sa mga installer na na-download mula sa isang cloud drive: kunin ang checksum na ini-publish ng opisyal na pahina at ihambing ito sa halagang nakalkula dito. Kung kailangan mong i-download muli ang kliyente, magtungo sa Pahina ng pag-download ng Quark NetDisk PC para makuha ang installer.

Paano gumagana ang lokal na hashing

Nagpapadala ang mga modernong browser ng interface ng Web Crypto (crypto.subtle.digest) na nagko-compute ng SHA-1, SHA-256 at SHA-512 sa iyong device. Ang MD5 ay hindi bahagi ng interface na iyon, kaya ibinibigay ito ng isang open-source na pagpapatupad ng JavaScript na binuo sa pahina. Ang file ay binabasa sa memorya sa pamamagitan ng FileReader at na-hash sa mga tipak, na walang kahilingan sa network sa anumang punto.

Paano i-verify ang pag-download gamit ang checksum

Pagkatapos mag-download ng file mula sa cloud drive o opisyal na site, i-drop ito sa tool sa itaas para makuha ang checksum nito, pagkatapos ay ikumpara ito digit sa pamamagitan ng digit sa opisyal na halaga na inilista ng publisher. Ang eksaktong tugma ay nangangahulugan na ang file ay kumpleto at hindi nababago; ang isang solong magkaibang character ay nangangahulugan na ito ay nasira sa transit o binago, kaya i-download ito muli. Mas gusto ang SHA-256: Mabilis ang MD5 at SHA-1 ngunit hindi na angkop para sa tamper checking.

MD5 vs SHA-1 vs SHA-256: alin ang gagamitin

Ang lahat ng apat na algorithm ay gumagawa ng fixed-length na fingerprint, ngunit ang kanilang seguridad ay naiiba nang husto. Ang talahanayan sa ibaba ay nagbubuod ng mga nai-publish na pamantayan at mga abiso sa awtoridad.

FAQ

Ligtas bang mag-upload ng file sa site na ito para i-hash ito?
Walang ina-upload ang tool. Ang lahat ng pagkalkula ay lokal na tumatakbo sa iyong browser gamit ang Web Crypto at isang built-in na script, ang file ay hindi kailanman umaalis sa iyong device, at kahit na ito ay gumagana offline.
Bakit inaalok pa rin ang MD5 at SHA-1 kung hindi sila secure?
Ang mga ito ay mabilis at maayos para sa paghuli ng hindi sinasadyang katiwalian, ngunit ang mga banggaan ay maaaring sadyang gawin, kaya hindi dapat gamitin ang mga ito para sa tamper checking. Gamitin ang SHA-256 kapag mahalaga ang seguridad.
Bakit kayang kalkulahin ng browser ang SHA-256 ngunit hindi ang MD5?
Ang browser na Web Crypto interface ay nagpapadala lamang ng SHA-1/256/384/512, hindi MD5. Ang MD5 sa page na ito ay nagmula sa isang open-source na pagpapatupad ng JavaScript na lokal din na tumatakbo.
Ang aking halaga ay iba kung sakaling sa opisyal. May laban pa ba?
Ang mga hash ay case-insensitive at ang hex na content lang ang mahalaga. Kapag naghahambing, binabalewala ng tool ang case at space, kaya ang magkaparehong pagkakasunud-sunod ng character ay binibilang bilang isang tugma.
Paano kung hindi tugma ang checksum?
Nangangahulugan ito na ang file ay nasira sa panahon ng pag-download o binago. Tanggalin ito at i-download muli, mas mabuti mula sa opisyal na pinagmulan, pagkatapos ay i-verify muli.
Ano ang pagkakaiba ng GB at GiB?
Ang GB ay decimal (base 1000) at ang GiB ay binary (base 1024, ang IEC standard). Ang 1 GiB ay humigit-kumulang 1.074 GB, na isang dahilan kung bakit naiiba ang na-advertise na kapasidad sa ipinapakita ng system.
Maaari ko bang i-verify ang mga file na na-download mula sa Quark NetDisk?
Oo. Pagkatapos mag-download mula sa Quark NetDisk, i-drop ang file sa tool na ito para makuha ang SHA-256 nito at ikumpara ito sa checksum na ibinibigay ng publisher para kumpirmahin na kumpleto at mapagkakatiwalaan ang pag-download.
