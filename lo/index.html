File Hash Checker (ແລ່ນຢູ່ໃນທ້ອງຖິ່ນ, ບໍ່ມີການອັບໂຫລດ)

ວາງໄຟລ໌ໄວ້ແລ້ວບຼາວເຊີຂອງເຈົ້າຈະຄິດໄລ່ເຊັກຊັມ MD5 / SHA-1 / SHA-256 / SHA-512 ຂອງມັນຢູ່ໃນອຸປະກອນຂອງທ່ານ, ດັ່ງນັ້ນທ່ານສາມາດຢືນຢັນວ່າການດາວໂຫຼດສໍາເລັດ ແລະບໍ່ມີຄວາມເສຍຫາຍ ຫຼືຖືກລົບກວນ. ທຸກສິ່ງທຸກຢ່າງເຮັດວຽກຢູ່ໃນທ້ອງຖິ່ນ: ໄຟລ໌ບໍ່ເຄີຍອອກຈາກອຸປະກອນຂອງທ່ານແລະບໍ່ເຄີຍຖືກສົ່ງໄປຫາເຄື່ອງແມ່ຂ່າຍໃດໆ.

ນີ້ແມ່ນເປັນປະໂຫຍດໂດຍສະເພາະສໍາລັບຜູ້ຕິດຕັ້ງທີ່ດາວໂຫລດມາຈາກ cloud drive: ເອົາ checksum ທີ່ຫນ້າເວັບຢ່າງເປັນທາງການເຜີຍແຜ່ແລະປຽບທຽບກັບມູນຄ່າທີ່ຄິດໄລ່ຢູ່ທີ່ນີ້. ຖ້າທ່ານຕ້ອງການດາວໂຫລດລູກຄ້າອີກເທື່ອຫນຶ່ງ, ໃຫ້ໄປທີ່ຫົວຂໍ້ ຫນ້າດາວໂຫລດ Quark NetDisk PC ເພື່ອເອົາຕົວຕິດຕັ້ງ.

ວິທີການ hashing ທ້ອງຖິ່ນເຮັດວຽກ

ຕົວທ່ອງເວັບທີ່ທັນສະໄຫມສົ່ງສ່ວນຕິດຕໍ່ເວັບ Crypto (crypto.subtle.digest) ທີ່ຄິດໄລ່ SHA-1, SHA-256 ແລະ SHA-512 ໃນອຸປະກອນຂອງທ່ານ. MD5 ບໍ່ແມ່ນສ່ວນຫນຶ່ງຂອງການໂຕ້ຕອບນັ້ນ, ດັ່ງນັ້ນມັນຖືກສະຫນອງໂດຍການປະຕິບັດ JavaScript ແບບເປີດຢູ່ໃນຫນ້າ. ໄຟລ໌ໄດ້ຖືກອ່ານຢູ່ໃນຫນ່ວຍຄວາມຈໍາໂດຍຜ່ານ FileReader ແລະຖືກ hashed ເປັນ chunks, ໂດຍບໍ່ມີການຮ້ອງຂໍເຄືອຂ່າຍຢູ່ໃນຈຸດໃດ.

ວິທີການກວດສອບການດາວໂຫຼດດ້ວຍ checksum

ຫຼັງຈາກການດາວໂຫຼດໄຟລ໌ຈາກ cloud drive ຫຼືເວັບໄຊທ໌ທາງການ, ວາງມັນໃສ່ເຄື່ອງມືຂ້າງເທິງເພື່ອເອົາ checksum ຂອງມັນ, ຫຼັງຈາກນັ້ນປຽບທຽບມັນໂດຍຕົວເລກກັບມູນຄ່າຢ່າງເປັນທາງການຂອງລາຍຊື່ຜູ້ເຜີຍແຜ່. ການຈັບຄູ່ທີ່ແນ່ນອນຫມາຍຄວາມວ່າໄຟລ໌ແມ່ນສົມບູນແລະບໍ່ຖືກຂັດ; ຕົວອັກສອນທີ່ແຕກຕ່າງກັນຕົວດຽວຫມາຍຄວາມວ່າມັນເສຍຫາຍໃນລະຫວ່າງການໂດຍສານຫຼືຖືກແກ້ໄຂ, ສະນັ້ນດາວໂຫລດມັນອີກເທື່ອຫນຶ່ງ. ຕ້ອງການ SHA-256: MD5 ແລະ SHA-1 ແມ່ນໄວແຕ່ບໍ່ເຫມາະສົມສໍາລັບການກວດສອບການບິດເບືອນ.

MD5 vs SHA-1 vs SHA-256: ທີ່ຈະໃຊ້

ທັງສີ່ລະບົບສ້າງລາຍນິ້ວມືທີ່ມີຄວາມຍາວຄົງທີ່, ແຕ່ຄວາມປອດໄພຂອງພວກມັນແຕກຕ່າງກັນຢ່າງຫຼວງຫຼາຍ. ຕາຕະລາງຂ້າງລຸ່ມນີ້ສະຫຼຸບມາດຕະຖານທີ່ເຜີຍແຜ່ແລະແຈ້ງການອະນຸຍາດ.

ເຄື່ອງມືອັບໂຫລດບໍ່ມີຫຍັງ. ການຄຳນວນທັງໝົດເຮັດວຽກຢູ່ໃນບຼາວເຊີຂອງທ່ານໂດຍໃຊ້ Web Crypto ແລະສະຄຣິບໃນຕົວ, ໄຟລ໌ຈະບໍ່ອອກຈາກອຸປະກອນຂອງທ່ານ, ແລະມັນຍັງເຮັດວຽກແບບອອບໄລນ໌.
ເປັນຫຍັງ MD5 ແລະ SHA-1 ຈຶ່ງຍັງສະເໜີໃຫ້ຖ້າພວກເຂົາບໍ່ປອດໄພ?
ພວກເຂົາເຈົ້າແມ່ນໄວແລະປັບໄຫມສໍາລັບການຈັບການສໍ້ລາດບັງຫຼວງໂດຍບັງເອີນ, ແຕ່ການປະທະກັນສາມາດສ້າງໂດຍເຈດຕະນາ, ດັ່ງນັ້ນພວກເຂົາຕ້ອງບໍ່ຖືກນໍາໃຊ້ໃນການກວດສອບການຂັດຂວາງ. ໃຊ້ SHA-256 ເມື່ອຄວາມປອດໄພສຳຄັນ.
ເປັນຫຍັງຕົວທ່ອງເວັບສາມາດຄິດໄລ່ SHA-256 ແຕ່ບໍ່ແມ່ນ MD5?
ຕົວທ່ອງເວັບຂອງ Web Crypto interface ສົ່ງພຽງແຕ່ SHA-1/256/384/512, ບໍ່ແມ່ນ MD5. MD5 ໃນຫນ້ານີ້ແມ່ນມາຈາກການປະຕິບັດ JavaScript ແບບເປີດທີ່ເຮັດວຽກຢູ່ໃນທ້ອງຖິ່ນ.
ມູນຄ່າຂອງຂ້ອຍແຕກຕ່າງກັນໃນກໍລະນີທີ່ເປັນທາງການ. ມັນຍັງເປັນການແຂ່ງຂັນບໍ?

ມັນຫມາຍຄວາມວ່າໄຟລ໌ເສຍຫາຍໃນລະຫວ່າງການດາວໂຫລດຫຼືປ່ຽນແປງ. ລຶບມັນອອກແລ້ວດາວໂຫລດອີກຄັ້ງ, ໂດຍວິທີທາງການຈາກແຫຼ່ງທີ່ເປັນທາງການ, ຈາກນັ້ນໃຫ້ກວດສອບອີກຄັ້ງ.
ຄວາມແຕກຕ່າງລະຫວ່າງ GB ແລະ GiB ແມ່ນຫຍັງ?
GB ແມ່ນອັດຕານິຍົມ (ຖານ 1000) ແລະ GiB ແມ່ນຖານສອງ (ຖານ 1024, ມາດຕະຖານ IEC). 1 GiB ແມ່ນປະມານ 1.074 GB, ເຊິ່ງເປັນເຫດຜົນຫນຶ່ງທີ່ຄວາມສາມາດໂຄສະນາແຕກຕ່າງຈາກສິ່ງທີ່ລະບົບສະແດງໃຫ້ເຫັນ.
ຂ້ອຍສາມາດກວດສອບໄຟລ໌ທີ່ດາວໂຫລດມາຈາກ Quark NetDisk ໄດ້ບໍ?
ແມ່ນແລ້ວ. ຫຼັງຈາກການດາວໂຫຼດຈາກ Quark NetDisk, ວາງໄຟລ໌ເຂົ້າໄປໃນເຄື່ອງມືນີ້ເພື່ອເອົາ SHA-256 ຂອງມັນແລະປຽບທຽບກັບ checksum ທີ່ຜູ້ຈັດພິມສະຫນອງເພື່ອຢືນຢັນວ່າການດາວໂຫຼດແມ່ນສໍາເລັດແລະເຊື່ອຖືໄດ້.

ແຫຼ່ງຂໍ້ມູນ

ຂໍ້ເທັດຈິງທາງດ້ານວິຊາການໃນຫນ້ານີ້ແມ່ນລວບລວມຈາກການອ້າງອີງສາທາລະນະທີ່ມີອໍານາດຕໍ່ໄປນີ້:

  • MDN — Web Crypto API: SubtleCrypto.digest()
  • NIST FIPS 180-4 — Secure Hash Standard (SHA)
  • RFC 1321 — The MD5 Message-Digest Algorithm
  • RFC 6151 — Updated Security Considerations for MD5
  • NIST — NIST Retires SHA-1 Cryptographic Algorithm (2022)
  • NIST — Definitions of the SI units: binary prefixes
ຂໍ້ເທັດຈິງດ້ານວິຊາການທີ່ລວບລວມຈາກ MDN Web Crypto, NIST FIPS 180-4, RFC 1321 / RFC 6151 ແລະບັນທຶກ NIST ກ່ຽວກັບຫນ່ວຍສອງ. ນີ້ແມ່ນເຄື່ອງມືເປີດແຫຼ່ງສໍາລັບການກວດສອບຄວາມຖືກຕ້ອງຂອງໄຟລ໌ເທົ່ານັ້ນ. ບັນນາທິການ: Shen Yan.
