本地哈希是怎么算的

现代浏览器内置 Web Crypto 接口（crypto.subtle.digest），可在本机直接计算 SHA-1、SHA-256、SHA-512；MD5 不在该接口内，由页面内置的开源 JavaScript 实现完成。文件通过 FileReader 在内存中读取后逐块计算，整个过程不发起任何网络请求。

怎么用校验值核对下载文件

从网盘或官网下载文件后，把它拖进上面的工具算出校验值，再和发布方公布的官方校验值逐位比对：完全相同说明文件完整、未被篡改；只要有一位不同，就说明文件在传输中损坏或被人动过，应当重新下载。优先使用 SHA-256：MD5、SHA-1 算得快但已不适合做防篡改校验。

MD5、SHA-1、SHA-256 怎么选

四种算法都能产出固定长度的「指纹」，但安全强度差别很大。下表按公开标准与权威公告归纳。

FAQ

上传文件到这个网站算哈希安全吗？
工具不上传任何文件。计算全部在你浏览器本地用 Web Crypto 与内置脚本完成，文件不会离开你的设备，断网也能正常使用。
为什么 MD5 和 SHA-1 还能用，却说不安全？
它们算得快，适合检查文件有没有意外损坏；但已被证明能人为构造碰撞，不能再用于防篡改校验。需要安全性时请用 SHA-256。
为什么浏览器能算 SHA-256 却不能算 MD5？
浏览器的 Web Crypto 接口只内置了 SHA-1/256/384/512，没有 MD5。本页的 MD5 由开源 JavaScript 实现，同样在本地运行。
算出来的校验值和官方的大小写不一样，算一致吗？
哈希值不区分大小写，只看十六进制内容。比对时本工具会自动忽略大小写和空格，只要字符序列相同就算一致。
校验值对不上怎么办？
说明文件在下载中损坏或被改动过，建议删除后重新下载，最好换官方下载地址再校验一次。
GB 和 GiB 有什么区别？
GB 按 1000 进制（十进制），GiB 按 1024 进制（二进制，IEC 标准）。1 GiB 约等于 1.074 GB，这也是「标称容量」和系统显示容量有差的原因之一。
能校验夸克网盘下载的文件吗？
可以。从夸克网盘下载文件后，把它拖进本工具算出 SHA-256，再和发布方给出的校验值比对，即可确认下载是否完整可信。
