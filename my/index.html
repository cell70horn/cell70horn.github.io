File Hash Checker (စက်တွင်းအလုပ်လုပ်သည်၊ အပ်လုဒ်မလုပ်ပါ)

ဖိုင်တစ်ခုထဲထည့်လိုက်ပြီး သင့်ဘရောက်ဆာသည် ၎င်း၏ MD5/SHA-1/SHA-256/SHA-512 checksum ကို သင့်စက်ပစ္စည်းပေါ်တွင် မှန်ကန်စွာတွက်ချက်ပေးသည်၊ ထို့ကြောင့် သင်သည် ဒေါင်းလုဒ်တစ်ခုပြီးမြောက်ပြီး ပျက်စီးသွားခြင်း သို့မဟုတ် ဆော့ကစားထားခြင်းမရှိကြောင်း အတည်ပြုနိုင်ပါသည်။ အရာအားလုံးသည် စက်တွင်းတွင် လုပ်ဆောင်သည်- ဖိုင်သည် သင့်စက်မှ ဘယ်သောအခါမှ ထွက်ခွာမသွားဘဲ မည်သည့်ဆာဗာသို့မျှ ပေးပို့မည်မဟုတ်ပါ။

၎င်းသည် cloud drive မှဒေါင်းလုဒ်လုပ်ထားသော installers များအတွက် အထူးအဆင်ပြေပါသည်- တရားဝင်စာမျက်နှာထုတ်ဝေသည့် checksum ကိုယူပြီး ဤနေရာတွင်တွက်ချက်ထားသောတန်ဖိုးနှင့်နှိုင်းယှဉ်ပါ။ အကယ်၍ သင်သည် client ကို ထပ်မံဒေါင်းလုဒ်လုပ်ရန် လိုအပ်ပါက၊ အဆိုပါသို့သွားပါ။ Quark NetDisk PC ဒေါင်းလုဒ်စာမျက်နှာ installer ကိုရယူရန်။

1. ဖိုင် hash ဂဏန်းတွက်စက်

ဖိုင်တစ်ခုတွင် ရွေးပါ သို့မဟုတ် ဆွဲယူကာ checksum လေးခုစလုံးကို အလိုအလျောက်တွက်ချက်ပါသည်။ စာသားကို hash သို့ တိုက်ရိုက်ကူးထည့်နိုင်သည်။

သို့မဟုတ် ဖိုင်တစ်ခုကို ဤနေရာတွင် ချလိုက်ပါ - ၎င်းကို သင့်စက်ပေါ်တွင် လုပ်ဆောင်ပြီး မည်သည့်အခါမျှ အပ်လုဒ်မလုပ်ပါ။

2. အချိန်လွှဲပြောင်းခြင်းနှင့် ဒေတာယူနစ်ပြောင်းစက်

သတ်မှတ်အမြန်နှုန်းဖြင့် ဖိုင်တစ်ဖိုင်ကို လွှဲပြောင်းရန် အချိန်မည်မျှကြာမည်ကို ခန့်မှန်းပြီး ဒဿမ (GB) နှင့် ဒွိ (GiB) ယူနစ်များကြားသို့ ပြောင်းပါ။

ခန့်မှန်းခြေလွှဲပြောင်းချိန်—
ဒေတာယူနစ်ပြောင်းလဲခြင်း။

အစစ်အမှန်အမြန်နှုန်းသည် ဆာဗာ၊ လမ်းကြောင်းနှင့် ဆက်စပ်မှုပေါ်တွင်မူတည်သောကြောင့် ၎င်းသည် သီအိုရီအရ ခန့်မှန်းချက်သာဖြစ်သည်။

ဒေသန္တရ ဟက်ခ်လုပ်နည်း

ခေတ်မီဘရောက်ဆာများသည် သင့်စက်ပေါ်တွင် SHA-1၊ SHA-256 နှင့် SHA-512 ကိုတွက်ချက်ပေးသည့် Web Crypto interface (crypto.subtle.digest) ကို ပေးပို့သည်။ MD5 သည် ထိုအင်တာဖေ့စ်၏ အစိတ်အပိုင်းမဟုတ်သောကြောင့် ၎င်းကို စာမျက်နှာတွင် တည်ဆောက်ထားသော open-source JavaScript အကောင်အထည်ဖော်မှုဖြင့် ပံ့ပိုးပေးထားသည်။ ဖိုင်ကို FileReader မှတဆင့် Memory တွင်ဖတ်ပြီး မည်သည့်အချက်တွင်မျှ ကွန်ရက်တောင်းဆိုမှုမပါဝင်ဘဲ အတုံးလိုက်အတုံးလိုက်လုပ်ထားသည်။

Checksum ဖြင့် ဒေါင်းလုဒ်ကို ဘယ်လိုအတည်ပြုမလဲ။

cloud drive သို့မဟုတ် တရားဝင်ဆိုက်မှ ဖိုင်တစ်ခုကို ဒေါင်းလုဒ်လုပ်ပြီးနောက်၊ ၎င်း၏ checksum ကိုရယူရန် အထက်ပါကိရိယာထဲသို့ ချပေးပြီးနောက် ၎င်းကို ထုတ်ဝေသူစာရင်းများ၏ တရားဝင်တန်ဖိုးနှင့် ဂဏန်းများဖြင့် ဂဏန်းများဖြင့် နှိုင်းယှဉ်ပါ။ အတိအကျကိုက်ညီမှုဆိုသည်မှာ ဖိုင်သည် ပြီးပြည့်စုံပြီး တုန်လှုပ်ခြင်းမရှိသော၊ မတူညီသော စာလုံးတစ်လုံးသည် အကူးအပြောင်းတွင် ပျက်စီးသွားသည် သို့မဟုတ် မွမ်းမံထားသောကြောင့် ၎င်းကို ထပ်မံဒေါင်းလုဒ်လုပ်ပါ။ SHA-256 ကို ဦးစားပေးပါ- MD5 နှင့် SHA-1 တို့သည် မြန်ဆန်သော်လည်း ဆော့ကစားရန် မသင့်တော်တော့ပါ။

MD5 vs SHA-1 vs SHA-256- အသုံးပြုရန်

အယ်လဂိုရီသမ် လေးခုစလုံးသည် ပုံသေလက်ဗွေကို ထုတ်ပေးသော်လည်း ၎င်းတို့၏ လုံခြုံရေးမှာ သိသိသာသာ ကွဲပြားသည်။ အောက်ပါဇယားသည် ထုတ်ပြန်ထားသော စံနှုန်းများနှင့် တရားဝင်သတိပေးချက်များကို အကျဉ်းချုပ်ဖော်ပြထားသည်။

Algorithmအထွက်နည်းနည်းများသည်။လုံခြုံရေးအခြေအနေဘယ်အချိန်မှာ သုံးမလဲ။

FAQ

ဖိုင်တစ်ခုကို hash ရန် ဤဆိုက်သို့ အပ်လုဒ်လုပ်ခြင်းသည် ဘေးကင်းပါသလား။
ကိရိယာသည် ဘာမှမတင်ပါ။ တွက်ချက်မှုအားလုံးသည် Web Crypto နှင့် built-in script ကို အသုံးပြု၍ သင့်ဘရောက်ဆာတွင် စက်တွင်းအလုပ်လုပ်သည်၊ ဖိုင်သည် သင့်စက်မှ ဘယ်သောအခါမှ မထွက်ဘဲ၊ ၎င်းသည် အော့ဖ်လိုင်းတွင်ပင် အလုပ်လုပ်ပါသည်။
မလုံခြုံပါက MD5 နှင့် SHA-1 ကို အဘယ်ကြောင့် ကမ်းလှမ်းနေသေးသနည်း။
၎င်းတို့သည် မတော်တဆ အကျင့်ပျက်ခြစားမှုကို ဖမ်းမိခြင်းအတွက် လျင်မြန်ပြီး ကောင်းမွန်သော်လည်း တိုက်မိခြင်းများကို တမင်ဖန်တီးထားသောကြောင့် ၎င်းတို့ကို ခိုးယူစစ်ဆေးခြင်းအတွက် အသုံးမပြုရပါ။ လုံခြုံရေးအရေးကြီးသောအခါ SHA-256 ကိုသုံးပါ။
ဘရောက်ဆာသည် SHA-256 ကို အဘယ်ကြောင့် တွက်ချက်နိုင်သော်လည်း MD5 မဟုတ်ပါ။
ဘရောက်ဆာ Web Crypto interface သည် MD5 မဟုတ်ဘဲ SHA-1/256/384/512 ကိုသာ တင်ပို့သည်။ ဤစာမျက်နှာရှိ MD5 သည် ပြည်တွင်း၌လည်း လုပ်ဆောင်သည့် open-source JavaScript အကောင်အထည်ဖော်မှုမှ လာပါသည်။
ကျွန်ုပ်၏တန်ဖိုးသည် တရားဝင်တန်ဖိုးနှင့် ကွာခြားပါသည်။ တိုက်ဆိုင်နေသေးလား။
Hashes များသည် အသေးအမွှားကို အာရုံမခံဘဲ hex ပါဝင်မှုသာ အရေးကြီးသည်။ နှိုင်းယှဉ်သောအခါ၊ ကိရိယာသည် စာလုံးအသေးနှင့် space များကို လျစ်လျူရှုထားသောကြောင့် ထပ်တူထပ်မျှသော ဇာတ်ကောင်အစီအစဥ်ကို ကိုက်ညီမှုတစ်ခုအဖြစ် တွက်သည်။
checksum က မကိုက်ညီရင် ဘယ်လိုလုပ်မလဲ။
ဒေါင်းလုဒ်လုပ်နေစဉ် သို့မဟုတ် ပြောင်းလဲနေချိန်တွင် ဖိုင်ပျက်စီးသွားသည်ဟု ဆိုလိုသည်။ ၎င်းကိုဖျက်ပြီး ထပ်မံဒေါင်းလုဒ်လုပ်ပါ၊ အကောင်းဆုံးအားဖြင့် တရားဝင်အရင်းအမြစ်မှနေ၍ နောက်တစ်ကြိမ်စစ်ဆေးပါ။
GB နှင့် GiB ကွာခြားချက်ကဘာလဲ။
GB သည် ဒဿမ (အခြေခံ 1000) ဖြစ်ပြီး GiB သည် ဒွိ (အခြေခံ 1024၊ IEC စံနှုန်း) ဖြစ်သည်။ 1 GiB သည် 1.074 GB ခန့်ဖြစ်သည်၊ ယင်းသည် ကြော်ငြာထားသောစွမ်းရည်သည် စနစ်ပြသသည့်အရာနှင့် ကွဲပြားသည့်အကြောင်းရင်းတစ်ခုဖြစ်သည်။
Quark NetDisk မှဒေါင်းလုဒ်လုပ်ထားသောဖိုင်များကို အတည်ပြုနိုင်ပါသလား။
ဟုတ်ကဲ့။ Quark NetDisk မှ ဒေါင်းလုဒ်လုပ်ပြီးနောက်၊ ၎င်း၏ SHA-256 ကို ရယူရန် ဤကိရိယာထဲသို့ ဖိုင်ကို ချပေးပြီး ဒေါင်းလုဒ်သည် ပြီးပြည့်စုံပြီး ယုံကြည်စိတ်ချရကြောင်း ထုတ်ဝေသူမှ ပေးထားသည့် checksum နှင့် နှိုင်းယှဉ်ပါ။
