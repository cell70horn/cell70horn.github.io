檔案雜湊校驗工具（本地計算 · 不上傳）

把檔案拖進來，瀏覽器會在你本機算出它的 MD5 / SHA-1 / SHA-256 / SHA-512 校驗值，用來核對下載的檔案是否完整、有沒有損壞或被篡改。全程在本地運算，檔案不會離開你的裝置、不會上傳到任何伺服器。

校驗從雲盤下載的安裝包時尤其有用——拿到官方頁面公佈的校驗值，和這裡算出來的對一下即可。需要重新下載客戶端，可前往 夸克網盤電腦版官方下載頁 獲取安裝包。

本地雜湊是怎麼算的

現代瀏覽器內建 Web Crypto 介面（crypto.subtle.digest），可在本機直接計算 SHA-1、SHA-256、SHA-512；MD5 不在該介面內，由頁面內建的開源 JavaScript 實現完成。檔案透過 FileReader 在記憶體中讀取後逐塊計算，整個過程不發起任何網路請求。

怎麼用校驗值核對下載檔案

從網盤或官網下載檔案後，把它拖進上面的工具算出校驗值，再和釋出方公佈的官方校驗值逐位比對：完全相同說明檔案完整、未被篡改；只要有一位不同，就說明檔案在傳輸中損壞或被人動過，應當重新下載。優先使用 SHA-256：MD5、SHA-1 算得快但已不適合做防篡改校驗。

FAQ

上傳檔案到這個網站算雜湊安全嗎？
工具不上傳任何檔案。計算全部在你瀏覽器本地用 Web Crypto 與內建指令碼完成，檔案不會離開你的裝置，斷網也能正常使用。
為什麼 MD5 和 SHA-1 還能用，卻說不安全？
它們算得快，適合檢查檔案有沒有意外損壞；但已被證明能人為構造碰撞，不能再用於防篡改校驗。需要安全性時請用 SHA-256。
為什麼瀏覽器能算 SHA-256 卻不能算 MD5？
瀏覽器的 Web Crypto 介面只內建了 SHA-1/256/384/512，沒有 MD5。本頁的 MD5 由開源 JavaScript 實現，同樣在本地執行。
算出來的校驗值和官方的大小寫不一樣，算一致嗎？
雜湊值不區分大小寫，只看十六進位制內容。比對時本工具會自動忽略大小寫和空格，只要字元序列相同就算一致。
校驗值對不上怎麼辦？
說明檔案在下載中損壞或被改動過，建議刪除後重新下載，最好換官方下載地址再校驗一次。
GB 和 GiB 有什麼區別？
GB 按 1000 進位制（十進位制），GiB 按 1024 進位制（二進位制，IEC 標準）。1 GiB 約等於 1.074 GB，這也是「標稱容量」和系統顯示容量有差的原因之一。
能校驗夸克網盤下載的檔案嗎？
可以。從夸克網盤下載檔案後，把它拖進本工具算出 SHA-256，再和釋出方給出的校驗值比對，即可確認下載是否完整可信。
