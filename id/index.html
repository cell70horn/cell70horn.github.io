Pemeriksa Hash File (berjalan secara lokal, tidak ada unggahan)

Masukkan file dan browser Anda menghitung checksum MD5 / SHA-1 / SHA-256 / SHA-512 langsung di perangkat Anda, sehingga Anda dapat mengonfirmasi bahwa unduhan telah selesai dan tidak rusak atau diubah. Semuanya berjalan secara lokal: file tidak pernah meninggalkan perangkat Anda dan tidak pernah dikirim ke server mana pun.

Hal ini sangat berguna bagi installer yang diunduh dari drive cloud: ambil checksum yang diterbitkan halaman resmi dan bandingkan dengan nilai yang dihitung di sini. Jika Anda perlu mengunduh klien lagi, kunjungi Halaman unduh Quark NetDisk PC untuk mendapatkan penginstal.

Cara kerja hashing lokal

Browser modern mengirimkan antarmuka Web Crypto (crypto.subtle.digest) yang menghitung SHA-1, SHA-256, dan SHA-512 pada perangkat Anda. MD5 bukan bagian dari antarmuka tersebut, sehingga disediakan oleh implementasi JavaScript sumber terbuka yang ada di dalam halaman. File dibaca di memori melalui FileReader dan di-hash dalam beberapa bagian, tanpa permintaan jaringan kapan pun.

Cara memverifikasi unduhan dengan checksum

Setelah mengunduh file dari cloud drive atau situs resmi, masukkan file tersebut ke alat di atas untuk mendapatkan checksumnya, lalu bandingkan digit demi digit dengan nilai resmi yang dicantumkan penerbit. Pencocokan persis berarti file tersebut lengkap dan tidak diubah; satu karakter yang berbeda berarti rusak saat transit atau dimodifikasi, jadi unduh lagi. Lebih suka SHA-256: MD5 dan SHA-1 cepat tetapi tidak lagi cocok untuk pemeriksaan kerusakan.

FAQ

Apakah aman mengunggah file ke situs ini untuk melakukan hashing?
Alat ini tidak mengunggah apa pun. Semua komputasi berjalan secara lokal di browser Anda menggunakan Web Crypto dan skrip bawaan, file tidak pernah meninggalkan perangkat Anda, dan bahkan bekerja secara offline.
Mengapa MD5 dan SHA-1 masih ditawarkan jika tidak aman?
Mereka cepat dan baik untuk menangkap korupsi yang tidak disengaja, namun tabrakan dapat dibuat dengan sengaja, sehingga tidak boleh digunakan untuk pemeriksaan kerusakan. Gunakan SHA-256 jika masalah keamanan penting.
Mengapa browser dapat menghitung SHA-256 tetapi tidak dapat menghitung MD5?
Antarmuka Web Crypto browser hanya mengirimkan SHA-1/256/384/512, bukan MD5. MD5 di halaman ini berasal dari implementasi JavaScript sumber terbuka yang juga berjalan secara lokal.
Nilai saya berbeda dengan nilai resmi. Apakah masih cocok?
Hash tidak membedakan huruf besar-kecil dan hanya konten hex yang penting. Saat membandingkan, alat ini mengabaikan huruf besar dan spasi, sehingga urutan karakter yang identik dihitung sebagai kecocokan.
Bagaimana jika checksumnya tidak cocok?
Artinya file tersebut rusak saat diunduh atau diubah. Hapus dan unduh lagi, sebaiknya dari sumber resmi, lalu verifikasi sekali lagi.
Apa perbedaan antara GB dan GiB?
GB adalah desimal (basis 1000) dan GiB adalah biner (basis 1024, standar IEC). 1 GiB sama dengan 1,074 GB, yang merupakan salah satu alasan mengapa kapasitas yang diiklankan berbeda dari yang ditampilkan sistem.
Bisakah saya memverifikasi file yang diunduh dari Quark NetDisk?
Ya. Setelah mengunduh dari Quark NetDisk, masukkan file ke alat ini untuk mendapatkan SHA-256-nya dan bandingkan dengan checksum yang disediakan penerbit untuk mengonfirmasi bahwa unduhan telah selesai dan dapat dipercaya.
